# 首屏 30 秒 Wow 改造清单

目标：让用户在打开 App 的前 30 秒内，明确感受到“记得快、组织强、和普通笔记不一样”。

## P0

- [ ] 首页默认布局改成“可立即使用”的成品态
	- 默认首屏固定包含：快捷输入、最近记录、日历或时间轴、一个辅助组件（标签或地点）
	- 避免让新用户先面对空白 dashboard 或等待配置的工作台
	- 验收标准：用户打开后不需要理解 widget 系统，也不会觉得首页是空的

- [ ] 快捷输入做成首屏绝对主入口
	- 输入区域一眼可见，不需要用户寻找
	- 占位文案更像行动指令，而不是普通表单提示
	- 输入后反馈要明确、快速、克制
	- 验收标准：首次使用的人能在 3 秒内开始输入第一条记录

- [ ] 保存后增加联动反馈
	- 新记录保存后，最近记录要立刻更新
	- 日历、时间轴或对应组件要出现同步变化
	- 避免只有“已保存”提示，而没有系统联动感
	- 验收标准：用户能直观看到“内容进入系统并被组织”，而不是被丢进列表

## P1

- [ ] 新用户空状态改成“轻引导 + 轻示例”
	- 不要纯空白页
	- 提供 1 到 3 条像真实使用痕迹的示例内容
	- 空状态文案强调“随手记，之后能回看和整理”
	- 验收标准：新用户首次打开时能理解这个 App 的核心价值，而不是只看到空容器

- [ ] 做一个明确的品牌记忆点
	- 保留克制但可识别的鸭子品牌元素
	- 统一首页文案和视觉气质，避免既像工具页又像教程页
	- 动效只保留 1 到 2 个真正有感知的关键反馈点
	- 验收标准：用户用完后能记住这不是普通备忘录，而是“那个很会帮我记东西的 App”

## 判断标准

- [ ] 用户打开首页 5 秒内，能看懂这是“快速记录 + 自动组织”的产品
- [ ] 用户输入一句话后，10 秒内能看到内容出现在首页某个组件中
- [ ] 用户不会把首页误解为设置页、空白容器或仅支持列表记录的普通笔记 App
